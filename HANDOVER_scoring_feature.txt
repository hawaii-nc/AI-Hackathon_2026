WHAT THIS FEATURE DOES
---------------------------------------------------------------------------
For every patient (from the handwritten-note image scanner) and every shelter
(service), score how well they fit on two axes, save the results as two tables,
and let a case worker fire a Gemini-drafted referral email to a *selected*
shelter that cross-references the patient's info and the parameter match.

Pipeline:

  patient_raw (OCR text)   --extract-->  structured patient params
  services_parameters.csv  --load----->  structured service params
                     \                    /
                      \--- scoring.py ---/   per (patient x service):
                            confidence    = UNWEIGHTED mean of matched rules (0-100)
                            compatibility = WEIGHTED mean (PARAM_WEIGHTS)     (0-100)
                                  |
                                  v
              score_store.py  -> CSVs + .sql files + Supabase tables
                                  |
                                  v
      referral.py draft_patient_referral_email() -> Gemini email for chosen shelter

- confidence    = "how many of the parameters we could check actually line up"
                  (every rule counts equally).
- compatibility = same comparison, but each rule has an arbitrary importance
                  weight (gender/age/sobriety/DV ~9-10, pets ~3). Edit
                  PARAM_WEIGHTS in backend/app/services/scoring.py to tune.
- Each rule evaluates to 1.0 (match) / 0.0 (hard mismatch) / None (n/a or
  unknown -> skipped from BOTH scores). Rules live in scoring.RULES.

THE BLOCKER (read this first)
---------------------------------------------------------------------------
DDL (CREATE TABLE) cannot go through the `supabase` Python client or ANY API key
- not the anon key, not the service-role key. PostgREST (the data API) only does
rows/cells (select/insert/update/upsert), never table creation. This is NOT a
permission you can toggle.

To create tables from Python you must use ONE of two *different* credentials,
added to backend/.env. The code auto-detects whichever is present
(score_store.run_sql -> prefers DB URL, then Management API):

  Option A - SUPABASE_ACCESS_TOKEN  (Management API personal access token)
    * No install needed (uses httpx, already installed).
    * Account-wide token. Get at:
        https://supabase.com/dashboard/account/tokens
    * .env line:
        SUPABASE_ACCESS_TOKEN=sbp_xxxxxxxxxxxxxxxx

  Option B - SUPABASE_DB_URL  (direct Postgres connection URI)
    * Needs:  pip install psycopg2-binary
    * Project-scoped. Dashboard -> Settings -> Database -> Connection string -> URI.
    * .env line:
        SUPABASE_DB_URL=postgresql://postgres:<db-password>@db.krzfxnarxnhdjsqvjprg.supabase.co:5432/postgres
    * If db.<ref>.supabase.co hangs on Windows (IPv6-only on some projects), use
      the "Session pooler" URI shown on that same Database settings page instead.

As of this handover, backend/.env contains ONLY SUPABASE_URL and SUPABASE_ANON_KEY
- neither DDL credential, nor the service-role key, is in the file yet.

TABLE LAYOUTS (two options, both generated)
---------------------------------------------------------------------------
"matrix"  (backend/sql/score_seed.sql, tables: confidence / compatibility)
    rows = patients, columns = services. Column names are sanitized
    (maui_rescue_mission); the service_columns lookup table maps id -> human name.

"charts"  (backend/sql/compatibility_chart.sql, tables: "confidence chart" /
           "compatibility chart")
    rows = services, columns = profile names. This is the layout last requested
    (services down the first column, profile names as the header row).
    NOTE: a table name with a space MUST be double-quoted everywhere:
        select * from "compatibility chart";
    Unquoted is a syntax error (this is what made an earlier manual paste "fail").

CSV mirrors (always written, no network needed):
    backend/confidence_scores.csv, backend/compatibility_scores.csv
    (first column "patient", one column per service).
